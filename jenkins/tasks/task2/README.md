# Task 2: Declarative Pipeline with Parallel Tests and Notifications

## Overview

This Task 2 implementation uses the Node.js + Express sample app in `app/` and a Declarative Jenkins pipeline with required stage order:

1. Checkout
2. Build
3. Test (parallel: Unit + Integration)
4. Package
5. Deploy

Pipeline file for this task is:

- `jenkins/tasks/task2/Jenkinsfile`

The sample application is at:

- `app/`

## Requirement Mapping

- App under `app/` with REST endpoint: implemented (`GET /health` in `app/src/app.js`)
- Unit tests: 6 tests in `app/tests/unit/banking.test.js`
- Integration tests: 3 tests in `app/tests/integration/app.integration.test.js`
- Declarative Jenkinsfile with required stage order: implemented in `jenkins/tasks/task2/Jenkinsfile`
- Parallel test execution: `Unit Tests` and `Integration Tests` branches run in `parallel {}`
- JUnit publication per branch: each branch publishes its own JUnit XML report
- Fail pipeline if one branch fails: `failFast true` and normal Jenkins stage failure behavior
- Agent label: pipeline uses `agent { label 'linux-agent' }`
- Secret injection via `credentials()`: `SLACK_WEBHOOK = credentials('slack-webhook')`
- `post` handlers:
  - `always`: archives artifacts
  - `success`: sends Slack message including `BUILD_URL`
  - `failure`: sends Slack message including failing stage name

## Jenkins Prerequisites

Create this Jenkins credential before running the pipeline:

- ID: `slack-webhook`
- Type: `Secret text`
- Value: your Slack incoming webhook URL

Install/verify plugins:

- Pipeline
- JUnit
- Git
- GitHub Branch Source (for multibranch)
